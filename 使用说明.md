【租帮宝 - 使用说明】

一、功能简介
   1. 多站点监控：同时监控多个业务后台的“待处理订单”数量。
   2. 实时提醒：发现新订单时，支持桌面气泡 + 企业微信/飞书消息通知。
   3. 快速处理：双击列表项即可一键打开后台处理页面。
   4. 后台运行：支持最小化到托盘，关闭主窗口不退出程序。

二、站点配置说明
   在“站点管理”页添加或编辑站点。
   提示：在站点列表上点击鼠标右键，可以快速修改选中站点的配置。

   1. 站点名称
      用于区分不同的后台系统，建议使用简短易懂的名字，如“上海后台”、“北京后台”。

   2. 登录地址
      目标后台系统的登录页面网址（URL）。软件将自动打开此地址进行登录操作。

   3. 用户名 & 密码
      用于自动登录后台的账号信息。
      注意：建议为软件配置独立的子账号，避免与人工操作共用同一账号，防止互相顶号导致掉线。

   4. 订单页地址（待审核）
      登录成功后，软件需要监控的具体订单列表页面网址,可以看到待审核订单的页面地址。
      如果不填，软件会尝试自动寻找，但为了准确性，建议手动填写。

   5. 启用此站点监控
      勾选后，该站点才会参与轮询监控；取消勾选则暂时忽略该站点。

   6. 选择器配置 (JSON)
      高级选项，用于精确告诉软件如何定位页面上的输入框、按钮和订单数量。
      通常由技术人员配置，普通用户无需修改。如果遇到无法登录或无法识别订单的情况，请联系交付人员。

三、全局运行配置
   在“高级设置”页可以调整软件的运行行为：

   1. 监控间隔 (秒)
      软件每隔多久检查一次所有站点的订单状态。
      默认 60 秒。建议不要设置过短（如低于 30 秒），以免被目标网站识别为频繁请求而封禁。

   2. 开启桌面气泡通知
      勾选后，当检测到新订单时，电脑右下角会弹出气泡提醒。

   3. 无头模式 (隐藏浏览器)
      勾选后，浏览器将在后台静默运行，不会在桌面显示窗口。
      建议：仅在当天已成功完成一次登录（确保无需验证码）后开启。
      注意：如果遇到需要输入验证码或滑块的情况，必须关闭此模式，使用“显示浏览器界面”进行人工辅助。

   4. 夜间免打扰模式
      勾选后，软件将在指定的时间段内暂停发送通知，避免打扰休息。
      - 静默时段：设置开始时间和结束时间（24小时制）。例如设置 23 到 7，表示晚上 11 点到次日早上 7 点不提醒。

四、消息通知配置
   支持将订单消息推送到企业微信或飞书群聊：

   1. 企业微信 Webhook
      用于将通知发送到企业微信群。
      获取方式：企微群聊 -> 右上角设置(...) -> 消息推送 -> 添加 -> 复制 Webhook地址。
      将地址填入软件的“企业微信通知配置”中即可。

   2. 飞书 Webhook
      用于将通知发送到飞书群。
      获取方式：在飞书群聊 -> 群设置 -> 机器人 -> 添加机器人 -> 自定义机器人 -> 复制 Webhook 地址。

五、启动监控与日常操作
   1. 启动服务
      回到“运行监控”页，点击“启动监控服务”开始工作。列表会实时显示每个站点的待处理数量和更新时间。

   2. 处理订单
      双击列表中的站点，软件会自动打开浏览器并跳转到订单页面，方便快速处理。

   3. 人工介入
      如果遇到需要输入验证码或滑块的情况，点击“显示浏览器界面”，手动完成操作后，再点击“隐藏/移出屏幕”即可。

六、常见问题与注意事项

   1. 找不到软件窗口？
      检查电脑右下角的托盘图标（小图标区域），右键点击图标可以选择“显示主界面”或“退出”。

   2. 关于多账号防关联
      软件内置了环境隔离机制，不同站点的登录环境（Cookie、缓存）是独立的，互不干扰。

   3. 为什么显示 0 单？
      - 可能确实没有待处理订单。
      - 可能登录失效，请点击“显示浏览器界面”查看是否停留在登录页。

   4. 无法自动登录？
      - 检查登录地址、账号密码是否正确。
      - 如果页面有验证码，软件无法自动处理，需要人工介入。
      - 每天手动登录一次，主要原因是各平台的登录状态会定时过期，需要手动登录保持状态；一般一天一次登录即可，除非被顶号或者网络波动导致登录失效。

   5. 频繁提示需要人工登录
      建议每个站点使用独立的账号密码。如果多处共用同一账号，容易互相顶号导致反复掉线。
